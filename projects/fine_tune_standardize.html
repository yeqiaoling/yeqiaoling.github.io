LLM-powered Data Standardization

Fine-tuning GPT-4o-mini for Electronic Component Parameter Normalization

🔍 Project Motivation

Electronic component data often contains inconsistent and redundant parameters. Manual normalization is labor-intensive and error-prone. This project aims to automate parameter standardization using fine-tuned LLMs.

🛠️ Methodology

  • Prepared 300+ structured examples for fine-tuning (with prompt templates & labeling logic)
  • Fine-tuned gpt-4o-mini with OpenAI API
  • Deployed with FastAPI (endpoints: /batch_standardize and /health_check)

📊 Results

  • 40% redundant fields identified and standardized
  • Model inference latency < 1s via API
  • Flexible schema: stem/suffix/value/unit/symbol → unified output

🚀 Deployment

FastAPI Endpoint: /batch_standardize/
LLM-powered health check available at /health_check/

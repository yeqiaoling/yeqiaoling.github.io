LLM-powered Data Standardization

Fine-tuning GPT-4o-mini for Electronic Component Parameter Normalization

🔍 Project Motivation

Electronic component data often contains inconsistent and redundant parameters. Manual normalization is labor-intensive and error-prone. This project aims to automate parameter standardization using fine-tuned LLMs.

🛠️ Methodology

  • Prepared 300+ structured examples with prompt templates and labeling schema
  • Fine-tuned gpt-4o-mini via OpenAI’s fine-tuning API
  • Deployed the model using FastAPI with endpoints:
    • /batch_standardize: normalize parameter input
    • /health_check: real-time model and API availability

📊 Results

  • 40% redundant fields identified and consolidated
  • API latency: < 1s per request
  • Unified schema: stem + suffix + value + unit + symbol → standard format

🚀 Deployment

FastAPI Endpoints:
/batch_standardize/ – main inference API
/health_check/ – LLM-integrated monitoring
